---
name: 代码风格要求
description: 用户明确要求的编码风格：最少代码、禁止过度工程化、保留粗糙感
type: feedback
---

这是本科毕设项目，不追求生产级工程化。

**Why：** 用户明确说"不用追求事事完美，只要跑通大部分测试用例"。

**How to apply：**
- 用最少的代码完成目标，三行能搞定不写十行
- 禁止：过度抽象、helper函数封装一次性逻辑、设计模式、完善的错误处理、类型注解补全
- 禁止：feature flag、向后兼容shim、docstring补写、注释美化
- 允许：硬编码路径、全局变量、简单粗暴的实现，保留一定粗糙感
- 改动前先给方案，用户确认后再动手
